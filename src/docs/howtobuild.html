How to build SimTools packags

This file describes how to get source code and build libaries.
(for SimTools-1-01, Last update 2005-05-03)

Before start

We are using following environment to develop our packages.
  • Redhat 9, gcc 3.2.2 ( MacOS X is also used )
  • ROOT 4.03.04
  • Geant4.7.01 Patch1
  • JDK: j2sdk1.4.2_04
  • LCIO: v01-04
JDK and LCIO is not necessary, if you don't need outputs in LCIO format.

Get source code

The latest source code is available at our CVS sever, jlccvs.kek.jp. To checkout the latest one, do as follows. 
 $ cvs -d :pserver:anonymous@jlccvs.kek.jp:/home/cvs/soft login <ret>
 passwd: <ret>

  ( just enter blank to the password prompt. )

 $ cvs -d :pserver:anonymous@jlccvs.kek.jp:/home/cvs/soft co -P lcbase Leda lclib jsf physsim Jupiter Uranus Satellites
This will download the latest version of the source code in your current directory. To get source code corresponding to the version used for SimTools-1-01, do as follows. 
 $ cvs -d :pserver:anonymous@jlccvs.kek.jp:/home/cvs/soft co -P -r SimTools-1-01 lcbase Leda lclib jsf physsim Jupiter Uranus Satellites

Build package

Priar to start make, environment parameters must be set properly, as shown in the sample setup.bash file. Please edit this file to match with your host, then do 
   $ source setup.bash
Then following commands will build all package. 
 $ ( cd $LCBASEDIR ; make )
 $ ( cd $LEDAROOT ; make )
 $ ( cd $LCLIBROOT ; configure ; make all )
 $ ( cd $JSFROOT ; make install )
 $ ( cd $JUPITERROOT ; make )
 $ ( cd $URANUSROOT  ; make )
 $ ( cd $SATELLITESROOT ; make )
